# 守护智慧稻田游戏设计文档

日期：2026-05-26

## 1. 目标

《守护智慧稻田》是一款面向 1-5 年级小学生的农业科普互动游戏。玩家在真实稻田场景中观察水稻从插秧到成熟收割的变化，识别随机出现的 3 种病虫害，并选择正确工具完成防治。

游戏重点不是速度挑战，而是让学生通过点击、观察、尝试，建立三个基础认知：

1. 水稻有清晰的生长阶段。
2. 稻田病虫害有不同外观和名称。
3. 不同病虫害需要匹配不同防治工具。

## 2. 设计假设

- 首版为单人触屏网页游戏，适合平板、电脑浏览器和教室大屏演示。
- 目标学生年龄跨度较大，因此交互必须低门槛：看见目标、点工具、点病虫害。
- 游戏不设置倒计时，不做生命值惩罚，错误选择只提示重新选择。
- 视觉素材以真实照片为主：稻田、水稻、病虫害、工具都使用实景图。
- 仅“稻田守护员”使用卡通人物，作为学生的行动代理和亲和角色。
- 首版不追求真实农药剂量、复杂农艺流程或专业植保判断，只表达“识别-匹配-处理”的科普关系。
- 若真实病虫害照片视觉冲击较强，应选择清晰但不过度惊悚的教学图片。

## 3. 成功标准

学生完成一局后应能：

1. 按顺序说出水稻 5 个生长阶段中的至少 3 个。
2. 看到游戏中出现过的病虫害时，能读出其名称。
3. 能把出现过的病虫害和对应工具进行匹配。
4. 理解错误工具不会失败，可以重新尝试。

产品验收标准：

- 一局游戏只随机出现 3 种不同病虫害。
- 3 种病虫害全部清除后弹出通关界面。
- 错误工具不会清除病虫害，并显示“工具不对，请重新选择”。
- 五个水稻阶段按顺序推进，顶部进度条能明确显示当前阶段。
- 工具栏图片、文字、点击区域足够大，适合小学生手指点触。

## 4. 推荐实现范围

首版建议做成“固定流程 + 随机目标”的轻量教育游戏：

- 固定水稻阶段顺序：插秧期、分蘖期、拔节期、抽穗期、成熟收割期。
- 每局开局随机抽取 3 个不同病虫害。
- 病虫害在五个阶段推进过程中依次出现，也可以在同一阶段出现多个，但总数固定为 3。
- 玩家选择工具后点击病虫害目标，系统立即判定。
- 全部清除后显示通关总结。

不建议首版加入：

- 复杂背包系统。
- 多关卡地图。
- 时间限制。
- 分数排名。
- 真实药剂配比。
- 农田经营数值。

这些功能会增加认知负担，偏离“农业知识科普与互动体验”的目标。

### 4.1 下一阶段设计补充：五阶段关卡模式

2026-05-26 追加设计方向：

当前版本已经有五个水稻生长阶段的视觉背景，但玩法仍然像“一局随机抽 3 个目标”，学生不容易感受到“每个生长阶段会遇到不同问题”。下一阶段应把五个生长阶段升级成五个关卡，每个阶段都有自己的病虫害候选池和通关目标。

这里按“水稻”继续设计。用户描述中出现“麦子”一词，暂按口误处理；如果后续目标作物要改成小麦，需要重做生长期、病虫害和美术资产。

推荐采用两种模式：

1. 自动模式，作为默认模式。游戏从插秧期开始。玩家在当前阶段完成 3 个正确处理任务后，本阶段过关，弹出阶段结算，然后点击“下一步”进入下一个水稻阶段。每个阶段的背景、关卡标题、病虫害候选池都会变化。
2. 手动模式，作为自由练习模式。玩家可以在关卡选择界面直接选择任一阶段，例如“分蘖期”或“抽穗期”，单独练习该阶段的病虫害识别。

自动模式更适合课堂主线，能让学生顺着水稻生长过程学习。手动模式更适合复习、展示和错题练习。

### 4.2 阶段病虫害候选池

每个阶段不再只有 3 种病虫害，而是配置 4-7 种候选对象。每次进入阶段时，从该阶段候选池里随机抽取 3 种作为本轮任务。这样既保留低龄学生可以完成的任务量，又能让重复游玩出现变化。

阶段候选池应满足：

- 每个阶段候选对象 4-7 种，最多不超过 7 种。
- 每轮只出现 3 种任务目标。
- 3 种目标全部正确处理，即本阶段过关。
- 同一阶段重复游玩时可随机出现不同组合。
- 候选池需要兼顾真实农业知识和儿童可识别性，避免加入过于相似或难以区分的病害。

基于现有资料的教学简化候选池建议如下。该表不是最终农技处方，而是儿童游戏的知识结构草案：

| 水稻阶段 | 候选病虫害池 | 教学侧重点 |
|---|---|---|
| 插秧期 | 稻蓟马、稻飞虱、稻水象甲、螟虫、苗期叶瘟 | 小秧苗刚进入田里，重点观察嫩叶和幼苗受害 |
| 分蘖期 | 稻飞虱、稻纵卷叶螟、稻螟虫、纹枯病、叶瘟、稻叶蝉 | 植株变密，虫害和叶部病害更容易出现 |
| 拔节期 | 纹枯病、稻飞虱、稻纵卷叶螟、二化螟、稻瘟病、白叶枯病 | 稻秆长高，重点保护茎秆和叶片 |
| 抽穗期 | 穗颈瘟、稻曲病、纹枯病、稻飞虱、稻纵卷叶螟、螟虫、白叶枯病 | 稻穗开始形成，重点保护稻穗和上部叶片 |
| 成熟收割期 | 稻飞虱、稻曲病、穗颈瘟、纹枯病、稻纵卷叶螟、稻螟虫 | 稻谷成熟前，重点保护产量和稻穗品质 |

其中，现有首版已经有 6 个资产和规则：稻飞虱、稻螟虫、稻叶蝉、稻瘟病、纹枯病、稻曲病。下一阶段实现时，可以先复用这些对象，再逐步补充新对象素材，例如稻纵卷叶螟、稻蓟马、白叶枯病、穗颈瘟、稻水象甲。

### 4.3 工具选择次数与通关规则

每个阶段有 3 个随机任务目标，玩家最多可以选择 4 次工具。

规则：

1. 玩家每选择一次工具并点击一个病虫害，计为一次尝试。
2. 正确选择：清除该病虫害，正确数 +1。
3. 错误选择：显示“工具不对，请重新选择”，错误数 +1，目标保留。
4. 若 4 次尝试内完成 3 次正确处理，则本阶段过关。
5. 若 4 次尝试用完但正确数不足 3，则本阶段不通过，继续停留在当前阶段，可点击“再试一次”重新抽取或保留当前 3 个目标。
6. 自动模式下，本阶段过关后显示“下一步”按钮，点击后进入下一阶段。
7. 手动模式下，本阶段过关后显示“再玩本关”和“返回关卡选择”。

这个规则保留高容错，但加入轻微挑战：学生可以错 1 次，仍然能过关。相比无限尝试，它能让孩子更认真观察工具和病虫害的匹配关系。

### 4.4 阶段结算与下一步

阶段结算界面替代当前单一通关界面。

阶段过关时显示：

- 标题：本阶段守护成功！
- 当前阶段名称，例如“分蘖期”。
- 本阶段清除的 3 种病虫害。
- 每种病虫害对应的正确工具。
- 自动模式按钮：“下一步”。
- 手动模式按钮：“再玩本关”“返回关卡选择”。

五个阶段全部完成后，再显示最终通关界面：

- 标题：智慧稻田守护完成！
- 文案：你完成了水稻从插秧到成熟的全部守护任务。
- 展示五个阶段完成标记。
- 按钮：“重新开始”“选择关卡”。

### 4.5 调研依据与设计取舍

调研资料显示，水稻生育期可以划分为插秧、分蘖、拔节、抽穗、成熟等阶段；穗期防治资料常把穗期稻瘟病、稻曲病、稻纵卷叶螟、纹枯病、稻飞虱等作为主要对象；农技资料也常把螟虫、稻飞虱、稻纵卷叶螟和稻瘟病、纹枯病、稻曲病作为水稻重点病虫害防治对象。

参考资料：

- Bio-Protocol/Bio-101：水稻重要发育时期表型观察，https://cn.bio-protocol.org/mv2/e1010178
- 盐城市农业农村局：水稻穗期病虫害防治技术要点，https://snw.yancheng.gov.cn/art/2025/9/1/art_899_4362650.html
- 六安市农科院：水稻主要病虫害防治技术，https://www.laaas.cn/index.php/xinwendongtai/xingyezixun/202.html
- 玉溪市地方标准资料：水稻病虫害防治指标，https://m.yuxi.gov.cn/u/cms/yxszfxxgk/202407/18093151kems.pdf

游戏不会直接照搬农技防治剂量和复杂处方，而是把资料简化成儿童可理解的“阶段-病虫害-工具”关系。后续实现前，还需要确认每个新病虫害的儿童友好图片素材和工具映射。

## 5. 核心玩法流程

```text
开始游戏
  ↓
展示插秧期稻田
  ↓
阶段进度自动或按事件推进
  ↓
随机刷新病虫害
  ↓
玩家选择工具
  ↓
玩家点击病虫害
  ↓
判断工具是否匹配
  ├─ 正确：守护员移动到目标附近，播放处理动作，清除病虫害
  └─ 错误：显示“工具不对，请重新选择”，病虫害保留
  ↓
继续阶段推进和目标刷新
  ↓
3 种病虫害全部清除
  ↓
通关提示与知识回顾
```

## 6. 场景设计

### 6.1 主画面

主画面由三部分组成：

- 中心主区域：大片实景稻田，是病虫害出现和处理的主要区域。
- 侧边工具栏：展示所有防治工具，使用真实工具图片和大号中文名称。
- 顶部进度条：展示水稻当前生长阶段和五阶段顺序。

主区域应尽量保持真实农业场景质感，避免夸张童趣背景、卡通稻田、过度饱和颜色或装饰性元素。

### 6.2 水稻阶段

每个阶段使用不同稻田实景背景或同一背景的阶段化图片：

| 阶段 | 画面要求 | 教学重点 |
|---|---|---|
| 插秧期 | 水田中刚移栽的矮小秧苗，水面明显 | 水稻从秧苗开始 |
| 分蘖期 | 秧苗长出分枝，株丛变密 | 一株水稻会变得更繁茂 |
| 拔节期 | 稻秆节节拔高，植株明显长高 | 水稻进入快速长高阶段 |
| 抽穗期 | 稻株抽出稻穗，进入扬花阶段 | 稻穗开始形成 |
| 成熟收割期 | 稻穗饱满泛黄，稻谷完全成熟 | 水稻成熟，可以收割 |

阶段推进可以采用短暂淡入淡出切换，避免强烈闪烁。顶部进度条同步高亮当前阶段。

## 7. 角色设计

稻田守护员为唯一卡通角色：

- 身着标准农业工作服，可包含草帽、工作靴、手套等农业劳动元素。
- 站在稻田侧边或田埂上，不遮挡主要病虫害目标。
- 点击正确目标时，守护员移动到目标附近并播放简单处理动作。
- 点击错误工具时，守护员不移动或做轻微摇头动作，配合文字提示。

角色风格应“亲切、专业、干净”，不要设计成超级英雄、魔法师或夸张搞笑角色。

## 8. 病虫害与工具映射

每局从以下 6 种病虫害中随机抽取 3 种不同目标。

| 类型 | 病虫害 | 展示方式 | 正确工具/方法 |
|---|---|---|---|
| 虫害 | 稻飞虱 | 真实稻飞虱图片 + 名称标签 | 喷雾器（驱虫药液） |
| 虫害 | 稻螟虫 | 真实稻螟虫图片 + 名称标签 | 捕虫网 |
| 虫害 | 稻叶蝉 | 真实稻叶蝉图片 + 名称标签 | 粘虫板 |
| 病害 | 稻瘟病 | 真实稻瘟病叶片或稻穗图片 + 名称标签 | 杀菌喷雾器 |
| 病害 | 纹枯病 | 真实纹枯病植株图片 + 名称标签 | 除草耙 |
| 病害 | 稻曲病 | 真实稻曲病稻穗图片 + 名称标签 | 病害清理铲 |

病虫害出现时应有轻微动态提示，例如柔和描边、轻微缩放或小幅闪动。提示只用于引导观察，不应破坏实景风格。

## 9. 工具栏设计

工具栏固定显示全部 6 个工具：

1. 喷雾器（驱虫药液）
2. 捕虫网
3. 粘虫板
4. 杀菌喷雾器
5. 除草耙
6. 病害清理铲

每个工具项包含：

- 真实工具实物图片。
- 大号中文完整名称。
- 明显选中状态。
- 足够大的点击区域。

点击工具后，该工具进入选中状态。再次选择其他工具时，选中状态切换到新工具。

## 10. 交互规则

### 10.1 正确工具

当玩家选择正确工具并点击对应病虫害：

1. 守护员移动到目标附近。
2. 播放对应作业动作或简短处理特效。
3. 播放清除音效。
4. 病虫害从稻田中消失。
5. 清除计数增加。

### 10.2 错误工具

当玩家选择错误工具并点击病虫害：

1. 病虫害保留。
2. 显示提示：“工具不对，请重新选择”。
3. 播放轻微提示音。
4. 玩家可立即选择其他工具再次尝试。

错误不扣分，不中断游戏。

### 10.3 未选择工具

当玩家未选择工具就点击病虫害：

显示提示：“先选择一个工具吧”。

该提示不属于原始需求，但能减少低龄学生的困惑，建议加入。

## 11. 音效设计

音效只做轻量反馈：

- 点击工具：短促清脆音。
- 选择正确并清除：轻快完成音。
- 选择错误：温和提示音。
- 通关：简短庆祝音。

不使用嘈杂环境声、持续背景噪声或突然的大音量效果。教室环境下默认音量应偏低，并提供静音按钮。

## 12. 通关界面

通关界面显示：

- 标题：守护成功！
- 文案：你帮助稻田清除了 3 种病虫害。
- 本局清除列表：展示 3 个病虫害名称及对应工具。
- 按钮：再玩一次。

如果用于课堂，可以增加一句回顾问题：

“你还记得哪种病虫害要用哪个工具吗？”

## 14. 状态机

游戏状态建议保持简单：

| 状态 | 含义 |
|---|---|
| ready | 初始页面，等待开始 |
| playing | 游戏进行中 |
| feedback | 正误提示短暂展示 |
| stage-completed | 当前阶段 3 个目标全部清除，等待进入下一阶段 |
| failed | 当前阶段 4 次尝试用完且未完成 3 次正确处理 |
| completed | 五个阶段全部完成，最终通关 |

核心运行数据：

- 当前生长阶段。
- 当前模式：自动模式或手动模式。
- 本局随机病虫害列表。
- 当前已出现病虫害。
- 当前已清除病虫害。
- 当前选中工具。
- 当前尝试次数。
- 当前正确处理次数。
- 当前提示文案。

## 15. 视觉与素材规范

### 15.1 实景图片

必须使用实景照片：

- 五个水稻生长阶段背景。
- 六种病虫害图片。
- 六种工具图片。

图片要求：

- 清晰、无遮挡、主体明显。
- 病虫害图片适合儿童观看。
- 风格尽量统一，不混用强烈滤镜、插画、3D 渲染图。
- 工具图片最好为白底或浅背景，便于在工具栏中识别。

### 15.2 卡通人物

守护员可以使用卡通风，但要和写实场景协调：

- 色彩不过度鲜艳。
- 边缘清晰。
- 动作简单。
- 不遮挡病虫害和工具文字。

## 16. 适配要求

主要适配：

- 平板横屏。
- 桌面浏览器。
- 教室大屏。

次要适配：

- 手机横屏。

建议横屏优先。若竖屏打开，提示用户横屏体验更好。

触控尺寸：

- 工具按钮建议不小于 88px 高。
- 病虫害点击热区应大于图片本身，可设置透明扩展区域。
- 顶部进度条文字需要在小屏下仍可读。

## 17. 测试与验收

功能测试：

1. 开始游戏后显示插秧期。
2. 阶段按五个顺序推进。
3. 自动模式下，当前阶段过关后点击“下一步”进入下一阶段。
4. 手动模式下，可以直接选择指定阶段练习。
5. 每个阶段候选池包含 4-7 种病虫害。
6. 每次进入阶段只随机出现 3 种任务目标。
7. 每种病虫害只接受对应工具。
8. 错误工具显示“工具不对，请重新选择”。
9. 未选择工具点击病虫害显示“先选择一个工具吧”。
10. 4 次尝试内完成 3 次正确处理，则当前阶段过关。
11. 4 次尝试用完但未完成 3 次正确处理，则停留当前阶段并允许再试。
12. 五个阶段全部完成后进入最终通关界面。

视觉测试：

1. 实景稻田占据主视觉。
2. 工具图片和文字清晰可读。
3. 病虫害目标不被角色、工具栏或进度条遮挡。
4. 选中工具状态明显。
5. 小屏横屏下文字不溢出。

课堂体验测试：

1. 一年级学生能在成人少量提示下完成一局。
2. 三年级以上学生能独立理解工具匹配规则。
3. 游戏节奏不制造焦虑。

## 18. 完整制作提示词

请制作一个名为《守护智慧稻田》的儿童农业科普互动游戏，面向 1-5 年级小学生。游戏整体采用实景写实风：农田、水稻、病虫害和工具都使用真实实拍图片；只有稻田守护员是卡通人物。画面贴近真实农业场景，不使用夸张童趣画风。

游戏主画面包含三部分：中心为大片实景稻田区域，侧边为固定工具栏，顶部为水稻生长阶段进度条。进度条依次展示插秧期、分蘖期、拔节期、抽穗期、成熟收割期，并高亮当前阶段。中心稻田画面要随进程逐步变化：插秧期为水田中刚移栽的矮小秧苗；分蘖期为秧苗长出分枝、株丛逐渐繁茂；拔节期为稻秆节节拔高、植株明显长高；抽穗期为稻株抽出稻穗、进入扬花阶段；成熟收割期为稻穗饱满泛黄、稻谷完全成熟。

设计一个卡通稻田守护员，身着标准农业工作服，站在稻田侧边或田埂上。玩家正确处理病虫害时，守护员移动到目标附近并做简单作业动作。角色要亲切、专业、干净，不遮挡稻田主体。

游戏中共有 6 种病虫害素材，全部用真实实景图片展示，并配中文名称标签：稻飞虱、稻螟虫、稻叶蝉、稻瘟病、纹枯病、稻曲病。每局游戏从这 6 种里随机选择 3 种不同病虫害，在水稻生长阶段推进过程中随机出现在稻田不同位置。病虫害出现时有轻微动态提示，例如柔和描边、轻微缩放或小幅闪动。

侧边工具栏固定展示 6 个工具，每个工具使用真实实物图片，并在图片下方用大号清晰汉字标注完整名称：喷雾器（驱虫药液）、捕虫网、粘虫板、杀菌喷雾器、除草耙、病害清理铲。工具排列整齐，点击区域大，适合手指点触。点击工具后显示明显选中状态。

病虫害和工具的正确对应关系如下：稻飞虱对应喷雾器（驱虫药液）；稻螟虫对应捕虫网；稻叶蝉对应粘虫板；稻瘟病对应杀菌喷雾器；纹枯病对应除草耙；稻曲病对应病害清理铲。

玩家操作方式为纯指尖点触：先点击工具，再点击稻田里的病虫害。若工具正确，守护员移动并处理，病虫害立即清除，播放轻快清除音效。若工具错误，弹出文字提示“工具不对，请重新选择”，病虫害保留，玩家可继续尝试。若玩家未选择工具就点击病虫害，提示“先选择一个工具吧”。游戏无严格时间限制，不设置失败惩罚，重点是农业知识科普和互动体验。

当本局随机出现的 3 种病虫害全部清除后，弹出通关界面。通关界面显示“守护成功！你帮助稻田清除了 3 种病虫害。”并列出本局清除过的 3 个病虫害及对应工具，同时提供“再玩一次”按钮。

音效保持简洁轻快：点击工具、清除病虫害、错误提示、通关分别使用短音效，不使用嘈杂声响。界面应适配平板横屏、桌面浏览器和教室大屏，文字清晰，按钮足够大，病虫害点击区域易于命中。

## 19. 仍需后续确认的非阻塞事项

以下事项不会影响设计方向，但会影响实现成本：

1. 实景图片来源：使用已有素材、公开授权图片，还是后续专门生成/拍摄。
2. 游戏运行平台：纯网页、本地离线 HTML，还是接入现有课程网站。
3. 阶段推进方式：自动按时间推进，还是清除目标后推进。
4. 是否需要教师讲解模式，例如显示每种病虫害的知识卡片。

首版建议先采用“清除目标后推进 + 通关回顾”的方式，保证学生能把操作结果和阶段变化联系起来。
